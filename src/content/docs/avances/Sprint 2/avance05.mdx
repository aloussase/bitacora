## Plan para terminar tareas pendientes

En primer lugar, se va a socializar con el cliente si intentar terminar con las
historias de usuario e implementar el proxy de deduplicación, o si continuar
haciendo pruebas de rendimiento con otras aplicaciones para tener resultados
más conclusivos acerca de las características del sistema desarrollado.

En el primer caso, se desarrollará un nuevo componente de OpenTelemetry, un
_receiver_, y se lo acoplará al colector customizado para lograr recuperar las
trazas deduplicadas antes de enviarlas a un determinado backend configurado, por
ejemplo, Grafana.

En el segundo caso, se instrumentará una o unas aplicaciones, en virtud del
tiempo, y se procederá a realizar pruebas de rendimiento de forma similar a
como se lo hizo con la aplicación QuickPizza. Esto permitirá evaluar el
comportamiento del sistema desarrollado de forma más general y permitirá tener
resultados más contundentes.

Sea cual fuere el camino a tomar, es posible completar el desarrollo en 2
semanas.
